IP Addresses

An IP (Internet Protocol) address is a unique numeric identifier assigned to every device on a network. These addresses allow devices to send and receive data across local networks and the internet. Understanding IP addresses is fundamental to DNS because DNS translates domain names into these IP addresses.

IPv4

IPv4 (Internet Protocol version 4) uses 32-bit addresses, typically written in dotted-decimal notation, like 192.168.0.1. Example range: 0.0.0.0 to 255.255.255.255.

  • Address Space: Approximately 4.3 billion addresses, which is nearly exhausted due to global internet growth.
  • Notation: Four octets (e.g., A.B.C.D), each ranging from 0 to 255.
  • Subnetting: Uses CIDR (Classless Inter-Domain Routing) to divide address blocks into smaller networks (e.g., 192.168.1.0/24).
  • Private Ranges: RFC 1918 defines addresses not routable on the public internet (e.g., 10.0.0.0/8, 192.168.0.0/16).

Subnetting (Overview)

Subnetting is the practice of dividing a larger IP network into smaller segments (subnets) to improve organization, security, or performance. 

# Example: Splitting 192.168.0.0/24 into two subnets
# 192.168.0.0/25  -> Hosts: 192.168.0.1 - 192.168.0.126
# 192.168.0.128/25 -> Hosts: 192.168.0.129 - 192.168.0.254

DNS must know which IP addresses belong to which domain or subdomain. With IPv4 running low, many networks rely on NAT (Network Address Translation) and private IP ranges.

IPv6

IPv6 (Internet Protocol version 6) is the successor to IPv4, using 128-bit addresses to vastly expand the available address space. A typical IPv6 address looks like 2001:0db8:85a3:0000:0000:8a2e:0370:7334.

  • Address Space: Supports 340 undecillion addresses (a virtually inexhaustible supply).
  • Shortened Notation: Leading zeros can be omitted, and consecutive groups of zeros can be replaced with "::" (e.g., 2001:db8:85a3::8a2e:370:7334).
  • Auto-configuration: IPv6 includes features like SLAAC (Stateless Address AutoConfiguration), making it easier to assign addresses without DHCP.
  • DNS Records: IPv6 addresses are stored in DNS as AAAA records (sometimes called “quad-A” records).

IPv6 Subnetting

Despite having a huge address pool, IPv6 also uses CIDR-like notations (e.g., 2001:db8:abcd:0012::/64). Subnetting in IPv6 is typically standardized around /64 blocks for most LAN segments.

Key Differences Between IPv4 and IPv6

Feature IPv4 IPv6
Address Size 32-bit 128-bit
Notation Dotted decimal (e.g., 192.168.0.1) Hexadecimal (e.g., 2001:db8::1)
Configuration Manual (DHCP, static), NAT often used Stateless auto-configuration, no NAT by design
DNS Records A records AAAA records
Exhaustion Nearly depleted Vast address pool

Many networks operate on a “dual-stack” basis, supporting both IPv4 and IPv6 simultaneously. DNS must resolve domain names to whichever protocol is in use.

Why IP Addresses Matter for DNS

  • Domain Name to IP Mapping: DNS is fundamentally about translating friendly names (like example.com) to numeric addresses (93.184.216.34 for IPv4 or 2606:2800:220:1:248:1893:25c8:1946 for IPv6).
  • Multiple IPs per Domain: Larger sites often use multiple IP addresses (for load balancing, redundancy).
  • Reverse DNS (PTR Records): IP addresses can map back to domain names via PTR records, useful for logging and security checks.
  • DNS & NAT: In IPv4, NAT can complicate DNS queries if internal addresses differ from external addresses.

Security & Best Practices

  • IPv4 NAT vs. IPv6 Security: IPv4 NAT is not a security feature per se, but it hides internal structures. IPv6 expects firewall-based filtering to protect devices.
  • IP Spoofing: Attackers may forge source IP addresses. Proper filtering (e.g., BCP38) helps mitigate this.
  • DNS Whitelisting/Blacklisting: Some security tools rely on specific IP-based rules. Ensure you manage changes to IP addresses carefully (especially with IPv6).

TL;DR

IP addresses are the fundamental building blocks of network communication. IPv4 is older, widely deployed, but nearly exhausted. IPv6 provides a massive pool of addresses and modern features. DNS supports both through A and AAAA records. Understanding IP address structure, subnetting, and differences between IPv4 and IPv6 is crucial for any network or DNS configuration.
