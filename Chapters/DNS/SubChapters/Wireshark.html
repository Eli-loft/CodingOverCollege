Introduction

Wireshark is a powerful, open-source network protocol analyzer used for capturing and inspecting network packets in real-time. It is a critical tool for network administrators, security analysts, and forensic investigators, enabling them to troubleshoot network issues, detect security threats, and analyze traffic flows in depth.

This chapter covers Wireshark's features, installation, capturing techniques, filtering methods, protocol analysis, troubleshooting techniques, automation, security implications, and best practices to ensure a thorough understanding of network packet analysis.

Installing Wireshark

Windows

  1. Download Wireshark from Wireshark.org.
  2. Run the installer and select Npcap during installation (required for packet capturing).
  3. Allow Wireshark to install WinPcap/Npcap for live traffic analysis.

Linux (Debian-based)

sudo apt update
sudo apt install wireshark -y

Grant non-root users permission to capture packets:

sudo usermod -aG wireshark $USER

Restart your system or log out and back in to apply changes.

macOS

brew install wireshark

Ensure Npcap is installed for proper packet capturing.

Understanding Wireshark’s Interface

Section Description
Menu Bar Provides access to file operations, view settings, capture options, and analysis tools.
Toolbar Contains shortcuts for starting/stopping captures, saving files, and applying filters.
Packet List Pane Displays captured packets in sequential order.
Packet Details Pane Shows protocol breakdowns for the selected packet.
Packet Bytes Pane Displays raw hexadecimal and ASCII data.
Status Bar Provides capture statistics and real-time updates.

Capturing Network Traffic

Starting a Packet Capture

  1. Open Wireshark.
  2. Select the network interface to monitor.
  3. Click Start Capture.
  4. Observe packets in real-time.
  5. Click Stop when enough data has been collected.

Saving Captures

Save captured data for later analysis:

File > Save As > Choose .pcapng format

Load a saved capture:

File > Open > Select .pcapng file

Filtering Packets for Analysis

Capture Filters (Before Capturing)

host 192.168.1.1      # Capture only traffic from/to this IP
port 80               # Capture only HTTP traffic
net 192.168.1.0/24    # Capture only traffic within a subnet

Display Filters (After Capturing)

ip.src == 192.168.1.1   # Show only packets from this source
ip.dst == 8.8.8.8       # Show only packets going to this destination
http.request            # Show only HTTP requests
udp.port == 53          # Show only DNS queries
Common Display Filters Description
tcp.port == 443 Filter HTTPS traffic
dns Display only DNS packets
icmp Show ICMP (ping) traffic
eth.addr == 00:11:22:33:44:55 Filter by MAC address
frame contains "password" Search for specific keywords in packets

Analyzing Protocols with Wireshark

HTTP/HTTPS Analysis

  • Identify GET, POST, and response codes.
  • Track cookies, authentication tokens, and API calls.
  • Filter web traffic using: http.request

DNS Analysis

  • Detect DNS queries and responses.
  • Identify malicious DNS exfiltration.
  • Filter DNS traffic using: dns && ip.addr == 8.8.8.8

TCP Handshake Analysis

  • Identify SYN, SYN-ACK, and ACK packets.
  • Troubleshoot failed connections.
  • Filter TCP handshake using: tcp.flags.syn == 1 && tcp.flags.ack == 0

ICMP (Ping) Traffic Analysis

  • Detect ping requests and responses.
  • Identify unreachable hosts.
  • Filter ICMP traffic using: icmp

Troubleshooting Network Issues

Issue Possible Cause Wireshark Filter
Slow Network Performance High latency or packet loss tcp.analysis.flags
DNS Resolution Failures Misconfigured DNS server dns && ip.addr == DNS_SERVER
Dropped Packets Firewall or congestion tcp.flags.reset == 1
Unauthorized Access Malicious or unknown IPs ip.src != TRUSTED_IP

Detecting Malicious Traffic

dns.qry.name contains "malware.com"
            
tcp.len > 1000 && ip.dst != TRUSTED_NETWORK

Automating Wireshark with Tshark

1. Capturing Traffic from CLI

tshark -i eth0 -w capture.pcapng

2. Applying Filters in Tshark

tshark -i eth0 -Y "http.request" -w http_traffic.pcapng

3. Extracting Specific Fields

tshark -r capture.pcapng -T fields -e ip.src -e ip.dst

Best Practices for Using Wireshark

  • Limit packet captures by using capture filters to reduce unnecessary data.
  • Use display filters efficiently to narrow down results.
  • Analyze traffic patterns and look for unusual spikes.
  • Save and document captures for future analysis.
  • Ensure compliance with organizational policies for network monitoring.

Conclusion

Wireshark is a powerful tool for analyzing, troubleshooting, and securing network traffic. By understanding packet structures, filtering techniques, protocol behaviors, and automation, users can gain deep insights into network performance and security threats.
