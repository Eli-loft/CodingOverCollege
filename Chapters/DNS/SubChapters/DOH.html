Introduction

The Domain Name System (DNS) is a critical component of the internet, translating human-friendly domain names into IP addresses. Traditional DNS queries, however, are sent in plaintextâ€”leaving them vulnerable to interception, spoofing, and censorship.

DNS over HTTPS (DoH) addresses these vulnerabilities by encrypting DNS queries within an HTTPS connection, ensuring that your DNS traffic remains private and secure.

What is DNS over HTTPS (DoH)?

Definition

DNS over HTTPS (DoH) is a protocol that encrypts DNS queries by transmitting them over an HTTPS connection instead of the traditional UDP/TCP (port 53). This encryption protects DNS data from eavesdropping and tampering.

Why Traditional DNS is Insecure

  • Unencrypted Queries: Allow interception and modification.
  • ISP Snooping: Service providers may log and misuse DNS data.
  • DNS Spoofing & Cache Poisoning: Attackers can inject false responses.
  • Censorship: Unencrypted DNS can be manipulated to block content.

How DoH Mitigates These Issues

  • Encrypts DNS traffic using TLS 1.3 over HTTPS (port 443).
  • Prevents unauthorized interception and modification of DNS queries.
  • Reduces the risk of DNS spoofing and cache poisoning.
  • Helps bypass censorship by blending DNS traffic with regular HTTPS traffic.

How DNS over HTTPS Works

Traditional vs. DoH Query Process

  1. Standard DNS Query:
    • User enters a domain (e.g., example.com) in a browser.
    • The browser sends an unencrypted DNS query to a resolver over UDP/TCP (port 53).
    • The resolver returns the IP address in plaintext.
  2. DoH Query:
    • The browser encrypts the DNS query using HTTPS/TLS.
    • The encrypted query is sent to a DoH resolver (e.g., Cloudflare 1.1.1.1 or Google DNS) over port 443.
    • The resolver decrypts the query, processes it, encrypts the response, and sends it back.
    • The browser receives the IP address securely.

Comparison: Traditional DNS vs. DoH

Component Traditional DNS DNS over HTTPS (DoH)
Transport Protocol UDP/TCP (Port 53) HTTPS (Port 443)
Encryption None TLS 1.3
Privacy Protection Vulnerable to snooping Encrypted and secure
Tamper Resistance Susceptible to spoofing Resistant to MITM attacks

DoH Query Format (JSON over HTTPS)

DoH queries are often structured in JSON and transmitted via HTTPS. Below is an example query and its corresponding response: 

{
  "name": "example.com",
  "type": 1
}
            
{
  "Status": 0,
  "Answer": [
    {"name": "example.com", "type": 1, "TTL": 299, "data": "93.184.216.34"}
  ]
}

Implementing DNS over HTTPS

Configuring DoH in Web Browsers

  • Google Chrome & Edge: Go to Settings > Privacy and Security > Security, scroll down to "Use Secure DNS", enable it, and select your preferred DoH resolver.
  • Mozilla Firefox: Open Settings > General > Network Settings, click "Settings", enable "Enable DNS over HTTPS", and choose a provider.

Configuring DoH in Operating Systems

  • Windows 11: Navigate to Settings > Network & Internet, edit DNS settings to "Encrypted Only (DNS over HTTPS)", and enter the DoH resolver addresses.
  • Linux (systemd-resolved): Edit /etc/systemd/resolved.conf to set your DNS server and enable DNSOverTLS=yes, then restart the service.

Running a Local DoH Proxy

You can deploy a local DoH proxy using tools like dnscrypt-proxy: 

sudo apt install dnscrypt-proxy
sudo nano /etc/dnscrypt-proxy/dnscrypt-proxy.toml
# Modify configuration:
server_names = ['cloudflare', 'google']
sudo systemctl restart dnscrypt-proxy

Security and Performance Considerations

  • Prevents DNS Hijacking: Encrypts queries to thwart man-in-the-middle attacks.
  • Enhances Privacy: Prevents ISPs and third parties from snooping on your DNS requests.
  • Harder to Block: Uses HTTPS (port 443), blending with regular web traffic.
  • Potential Latency: Encryption overhead may slightly increase query times.
  • Resolver Trust: Rely on reputable DoH providers to safeguard your data.

DoH vs. Alternative Secure DNS Protocols

Protocol Transport Encryption Port
DNS UDP/TCP None 53
DoH HTTPS TLS 1.3 443
DoT TCP TLS 1.3 853
DNSCrypt UDP/TCP Public Key Encryption 443

Best Practices for Deploying DoH

  • Use trusted DoH providers such as Cloudflare, Google, or Quad9.
  • Monitor DNS logs for anomalies with network monitoring tools.
  • Deploy local DoH proxies for enterprise-wide secure DNS resolution.
  • Combine DoH with DNSSEC to ensure both encryption and data integrity.

Conclusion and Next Steps

DNS over HTTPS (DoH) represents a significant improvement in securing DNS queries by encrypting them with TLS. This enhancement protects against interception, spoofing, and censorship while preserving user privacy.

Next Steps:

  • Enable DoH in your web browser and operating system.
  • Deploy a local DoH proxy if needed for enhanced control.
  • Regularly monitor your DNS traffic for unusual activity.
  • Consider integrating DoH with DNSSEC for a robust DNS security strategy.
