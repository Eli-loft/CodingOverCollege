Caching: Boosting Performance, One Hit at a Time

An in-depth look at how caching helps speed up applications, reduce latency, and cut down on unnecessary work.

Caching Introduction

In computing, caching is the process of storing data in a temporary storage area (the cache) so that future requests for that data can be served more quickly. Caches exist everywhere: in your CPU, your web browser, your operating system, your DNS resolver, and even distributed across global servers in Content Delivery Networks (CDNs).

Why Caching Matters

  • Speed: Serving cached data is much faster than computing or fetching it anew.
  • Reduced Latency: Resources stored locally (or closer to the user) reduce network round trips.
  • Lower Costs: Less bandwidth usage and fewer CPU cycles can mean big savings at scale.
  • Improved User Experience: Faster load times keep users engaged.

Types of Caches

1. CPU Cache

Tiny, ultra-fast memory built into modern processors (L1, L2, L3 caches) that speed up program execution by storing frequently accessed instructions and data.

2. Application Cache

Many server-side frameworks or client applications maintain their own in-memory caching layers (e.g., Redis, Memcached) to quickly serve results.

3. Database Cache

Databases often cache query results or frequently accessed records to reduce disk I/O and speed up responses.

4. DNS Cache

DNS resolvers cache domain-to-IP mappings to avoid repeated lookups. (See our previous chapter on Resolvers!)

5. Browser Cache

Modern browsers store images, CSS, JavaScript, and other assets locally to speed up page loads. HTTP headers like Cache-Control, Expires, ETag, and Last-Modified help manage this.

6. CDN (Content Delivery Network) Cache

CDNs store site content on servers across the globe. Users retrieve data from the nearest server, reducing latency and network congestion.

How Caching Works

  1. Request Data: The application or user requests a piece of data (e.g., a web page, an image, or a database record).
  2. Check Cache: Before doing the heavy lifting, the system checks if the data is already in the cache.
  3. Cache Hit or Miss:
    • Cache Hit: Data is found in the cache and returned immediately.
    • Cache Miss: Data isn’t in the cache, so the system fetches or computes it, then writes it to the cache.
  4. Expiration & Eviction: Data stays in the cache until it expires (TTL) or is evicted by an algorithm (LRU, LFU, FIFO) when space runs out or data is invalidated.

Common Caching Algorithms

  • LRU (Least Recently Used): Evicts the item that has not been used for the longest period of time.
  • LFU (Least Frequently Used): Evicts the item used the fewest times.
  • FIFO (First In First Out): Evicts the item that entered the cache earliest.
  • Random Replacement: Evicts a random item when needed.

These strategies aim to keep the most valuable or frequently requested data in cache for as long as possible.

Real-World Example: HTTP Caching

When you visit a website, the server can include HTTP headers like Cache-Control or ETag:

  • Cache-Control: max-age=3600 tells your browser it can reuse the resource for an hour.
  • ETag: A unique identifier for the resource. The browser can revalidate the resource by sending an If-None-Match header.

As a result, you see faster page loads on repeated visits or for shared resources across multiple pages.

Clearing Cache (Terminal Examples)

To manually clear different types of caches, use the following commands:


Windows (DNS Cache):

ipconfig /flushdns

macOS (DNS Cache):

sudo killall -HUP mDNSResponder

Linux (Systemd Resolver Cache):

sudo systemd-resolve --flush-caches

Pitfalls & Security

  • Stale Data: Overly aggressive caching can serve outdated information.
  • Inconsistency: Multiple layers of caches risk data inconsistency if invalidation is not carefully managed.
  • Privacy & Security: Sensitive data should not be cached publicly (e.g., personal user info). Use correct Cache-Control directives to ensure private data isn’t stored improperly.

TL;DR

Caching can drastically improve performance and reduce resource usage—just be mindful of eviction, expiration policies, and invalidation to avoid stale or inconsistent data. Whether you’re a systems designer, a web developer, or a curious learner, understanding caching is vital for building efficient, user-friendly applications.
