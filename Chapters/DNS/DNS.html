Chapter 1: Introduction to DNS

What is DNS?

Provides a clear definition of DNS and how it fits into the broader networking ecosystem. Explains the fundamental role DNS plays, comparing it to a “phone book” that maps human-readable domain names to IP addresses.

Basic Terminology

  • Domain names: (e.g., google.com).
  • IP addresses: Both IPv4 & IPv6 used by computers to locate each other.
  • DNS records: Various types (A, AAAA, CNAME, MX, etc.) that hold domain info.

By the end of this chapter, readers will understand why DNS is important, how it arose out of the need to make the internet more user-friendly, and the basic terms essential for further study.

Chapter 2: How DNS Works

DNS Resolvers

Differentiates between recursive and iterative resolvers. Demonstrates how a typical DNS query flows from a user’s computer to the DNS infrastructure, possibly going through multiple steps.

DNS Caching

Explores how caching improves performance by storing recent DNS lookups. Covers the importance of TTL (Time to Live) and how caching can both speed up and occasionally complicate DNS resolution.

Zone Files

Looks at how domain information is stored in zone files. Showcases common DNS record types in practice (A, AAAA, CNAME, MX, etc.).

This chapter connects definitions from Chapter 1 to the actual mechanics of DNS queries.

Chapter 3: Basic Networking Fundamentals

While DNS is a critical piece of the internet puzzle, it doesn’t exist in isolation. This chapter covers broader networking basics:

IP Addresses

Covers both IPv4 and IPv6 formats. Explains why IPv6 is increasingly important and how DNS integrates with it.

Protocols

Delves into UDP vs. TCP, explaining why DNS primarily uses UDP for queries and in which cases it might use TCP.

Ports

Highlights the well-known port numbers used by DNS (e.g., 53 for both TCP and UDP), plus how firewalls/routers handle DNS traffic.

Chapter 4: Public vs. Private DNS

Public DNS

Illustrates how public DNS providers (like Google’s 8.8.8.8 or Cloudflare’s 1.1.1.1) operate. Examines advantages like performance, reliability, and additional security features.

Private/Local DNS

Shows how to set up and maintain a local DNS server (beneficial for businesses or home labs). Discusses pros/cons: improved control, potentially faster lookups for internal services, etc.
